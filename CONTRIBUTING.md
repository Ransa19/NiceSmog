# Contributing to NiceSmog 🌍

Прежде всего, спасибо, что решили помочь проекту NiceSmog! Мы ценим любой вклад, который поможет жителям Алматы дышать более чистым воздухом.

В этом документе описаны правила и процессы участия в разработке.

---

## 🚀 Как вы можете помочь?

1.  **Разработка:** Исправление багов в визуализации карты или интеграция новых API (AirKaz, OpenAQ).
2.  **Данные:** Добавление новых координат датчиков в наш тестовый набор.
3.  **Документация:** Улучшение README, перевод инструкций или описание архитектуры.
4.  **Тестирование:** Проверка работы карты на разных мобильных устройствах и браузерах.

---

## 🛠 Процесс подачи изменений (Workflow)

Мы используем стандартную модель **Fork & Pull**:

1.  **Сделайте Fork** репозитория на свой аккаунт.
2.  **Создайте новую ветку** для ваших изменений:
    ```bash
    git checkout -b feature/название-вашей-фичи
    ```
3.  **Внесите изменения** и сделайте коммиты. Пожалуйста, пишите понятные сообщения к коммитам (например: `feat: add legend to heatmap`).
4.  **Отправьте изменения** в свой форк:
    ```bash
    git push origin feature/название-вашей-фичи
    ```
5.  **Откройте Pull Request (PR)** в основной репозиторий NiceSmog.

---

## 📝 Правила оформления кода и сообщений

- Используйте понятные названия переменных (мы используем Python/Folium).
- Если вы добавляете новую библиотеку, обязательно обновите файл `requirements.txt`.
- Старайтесь делать один PR для одной конкретной задачи.

---

## ⚖️ Этикет и правила поведения

Участвуя в этом проекте, вы обязуетесь соблюдать нормы нашего [Code of Conduct](./CODE_OF_CONDUCT.md). Мы выступаем за дружелюбное и инклюзивное общение.

## ❓ Есть вопросы?
Если вы не знаете, с чего начать, создайте новый **Issue** в GitHub с пометкой "question". Мы обязательно ответим!
